T2TOS Language – Syntax & Semantics Description

1. Overview

T2TOS is a command-driven, text-based scripting language designed for interactive execution in a sandboxed chat environment.
Every instruction is issued as a single line and is immediately evaluated.

The language is imperative, string-centric, and command-oriented, with lightweight support for variables, files, control flow, macros, and substitution.


2. Command Prefixes (Required)

Every valid instruction must begin with one of these prefixes:

Prefix

Meaning

/

Built-in command

$

Variable assignment or evaluation

@

File execution

Any line without these prefixes is rejected.


3. Tokens & Basic Syntax

Whitespace

  • Commands are space-separated
  • No line continuation
  • One command per line

Numbers

  • Integers only (unless implicitly converted)
  • Example: 5, -10

Symbols

Common symbols with special meaning:


< > $ = ; : [ ] ( ) { } % #


4. Substitution & Inline Expressions

T2TOS supports inline placeholders, evaluated before execution.

Chat History


<chat N>

  • Inserts the N-th chat entry (supports negative indexes)

History


<history N>

Random Numbers


<random MIN MAX>

Ranges


<range N>

Expands to:


0,1,2,...,N-1

File Field Access


<fileName.key>

  • Accesses a key from a map/ file


5. Variables

Assignment


$varName$value

$varName=value

Stored internally as:


var/value

Usage


<varName>

Reassignment

Overwrites the existing variable.


6. Filesystem Model

T2TOS uses an in-memory virtual filesystem.

File Types

Prefix

Type

var/

Variable

txt/

Plain text

lst/

List (semicolon separated)

map/

Key-value map

com/

Command list

sys/

System macro

fnc/

Function

lib/

Library


Saving Files


/file.save[content]filename

/file.save(content)filename

/file.save{content}filename

Opening Files


/file.open filename

Behavior depends on file type:

  • com/ → executes commands
  • lst/ → iterates items
  • fnc/ → parameterized execution
  • sys/ → macro expansion
  • txt/ → prints content


File Management


/file.delete filename

/file.deleteall

/file.copy(source)target

/file.name(old)new

/file.all

/desktop

8. Control Flow

Try / Catch


/try command :fails: fallback

If Statements (v2 only)


/if A:OP:B:command

Operators:


=  !=  >  <  >=  <=

Example:


/if 5:>:3:/echo yes


For Loops


/for x in a,b,c:command

Substitution:


<x>


9. Modes

Mode Switching


/mode.v1

/mode.v2

Mode

Features

Legacy syntax

v2

Enables /if, improved flow


10. Libraries & Globals

Libraries


/lib.save[libContent]libName

/lib.get libName

/lib.all

Global Variables


/global.set name value

/global.get name


11. Memory Register


/mem.in value

/mem.out

Acts as a single global scratch register.


12. Execution Shortcuts

Execute File


@filename

Execute Command File(com/, sys/ or fnc/)


/filename


13. Error Handling

  • Invalid commands raise sandbox errors
  • /try prevents crashes
  • /exit terminates execution


14. Summary

T2TOS is:

  • Line-based
  • Command-driven
  • Dynamically typed
  • Macro-heavy
  • Stateful (chat + files + globals)

It behaves like a hybrid of a shell, macro language, and chat scripting system.

Russian:

Язык T2TOS — описание синтаксиса

1. Общая информация

T2TOS — это текстовый командный язык, выполняемый в интерактивной песочнице (чат-среде).
Каждая строка является отдельной командой и выполняется сразу после ввода.

Язык:

  • императивный
  • ориентирован на команды
  • использует подстановки и макросы
  • хранит состояние (чат, файлы, переменные)


2. Обязательные префиксы команд

Каждая команда должна начинаться с одного из префиксов:

Префикс

Назначение

/

встроенная команда

$

работа с переменными

@

выполнение файла

Команды без этих префиксов считаются недопустимыми.


3. Лексика и базовый синтаксис

Разделители

  • аргументы разделяются пробелами
  • одна команда — одна строка
  • перенос строк не поддерживается

Числа

  • используются целые числа
  • пример: 10, -5

Специальные символы


< > $ = ; : [ ] ( ) { } % #


4. Подстановки (inline-выражения)

Подстановки вычисляются до выполнения команды.

История чата


<chat N>

Подставляет сообщение из чата (поддерживаются отрицательные индексы).

История команд


<history N>

Случайное число


<random MIN MAX>

Диапазон


<range N>

Результат:


0,1,2,...,N-1

Доступ к полю файла


<fileName.key>

Используется для файлов типа map/.


5. Переменные

Присваивание


$имя$значение

$имя=значение

Внутреннее представление:


var/значение

Использование


<имя>

Сохранение файлов

/file.save(содержимое)имя

/file.save{содержимое}имя

Открытие файлов

  • lst/ — перебирает элементы
  • fnc/ — функция с аргументами
